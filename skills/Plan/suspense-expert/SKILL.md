## 核心能力
- 悬疑推理世界观构建
- 线索铺陈设计
- 谜题结构设计
- 推理逻辑设计

## 线索系统

### 线索类型
```json
{
  "clue_types": [
    {"type": "physical", "description": "物理证据", "weight": 1.0},
    {"type": "testimonial", "description": "证人证词", "weight": 0.8},
    {"type": "behavioral", "description": "行为证据", "weight": 0.9},
    {"type": "circumstantial", "description": "间接证据", "weight": 0.6}
  ],
  "clue_density": "每3000字至少1个关键线索"
}
```

### 线索铺陈原则
- 每3000字至少1个关键线索
- 线索需要合理铺垫（不能凭空出现）
- 误导线索（红鲱鱼）不超过关键线索的2倍
- 所有线索最终需要形成完整证据链

## 谜题设计

### 谜题结构
```json
{
  "mystery_structure": {
    "surface_question": "表面问题",
    "deep_question": "深层问题",
    "clues": ["线索1", "线索2", "线索3"],
    "red_herrings": ["误导线索1"],
    "solution": "真相",
    "twist": "反转（可选）"
  }
}
```

### 经典谜题模板
1. **密室杀人** - 不可能的犯罪
2. **不在场证明** - 时间线造假
3. **身份替换** - 假死/伪装
4. **连环杀手** - 模式与签名
5. **消失的证据** - 销毁与隐藏

## 时间线设计
- 格式：年/月/日 时:分
- 精确时间戳
- 多时间线叙事时需清晰标注

## 氛围渲染
- 悬念钩子：每章结尾设置悬念
- 节奏控制：紧张与缓冲交替
- 视角选择：多视角叙事增加信息不对称

## 禁止出现
- 超自然解释（除非伏笔充分）
- 机械降神
- 线索未铺垫
- 逻辑漏洞

## 输出格式
生成悬疑推理相关世界观时，必须包含：
1. 核心谜题设计
2. 线索铺陈计划
3. 误导线索设计
4. 时间线结构
