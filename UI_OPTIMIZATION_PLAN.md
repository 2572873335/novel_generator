# NovelForge v4.2 UI 全面优化方案

## 一、整体布局重构

### 当前问题
1. 窗口大小 1500x850 在高分辨率屏幕上显得局促
2. 三栏比例 2:2:1 导致右侧情绪图过窄
3. 前期筹备室内容过多，单面板拥挤
4. 缺少顶部全局状态栏

### 窗口尺寸优化
- **主窗口**: 1600x900 (最小) → 支持 1920x1080 全屏
- **左侧 Agent 面板**: 固定 320px，可折叠
- **中间日志**: 自适应，最小 500px
- **右侧可视化**: 固定 360px

## 四、动效设计规范

### 1. 过渡动画
```css
/* 颜色过渡 */
transition: color 200ms ease,
            background-color 200ms ease,
            border-color 200ms ease,
            box-shadow 200ms ease;

/* 变换过渡 */
transition: transform 300ms cubic-bezier(0.4, 0, 0.2, 1);

/* 透明度过渡 */
transition: opacity 150ms ease;
```

### 2. 微交互动画
- **按钮悬停**: 发光效果 + 轻微上浮 (translateY(-2px))
- **卡片悬停**: 边框高亮 + 阴影加深
- **输入框聚焦**: 底部边框流光 (霓虹从左到右)
- **状态变化**: 颜色渐变过渡 (200ms)
- **Agent 激活**: 脉冲光环动画 (infinite)

### 3. 加载状态
- **骨架屏**: 脉冲渐变动画
- **进度条**: 霓虹流动效果
- **等待提示**: 旋转的赛博朋克风格 spinner

## 七、技术实现要点

### PyQt6 样式优化
```python
# 使用 QSS 变量 (通过 Python 字符串替换)
STYLE_TEMPLATE = """
QWidget {
    font-family: {font_family};
    font-size: {font_size};
}

QPushButton {{
    background: qlineargradient(
        x1: 0, y1: 0, x2: 0, y2: 1,
        stop: 0 {bg_light},
        stop: 1 {bg_medium}
    );
    border: 1px solid {border_color};
    border-radius: {radius}px;
    padding: {padding};
    color: {text_color};
    transition: all 200ms ease;
}}

QPushButton:hover {{
    border-color: {neon_cyan};
    box-shadow: 0 0 15px rgba(0, 245, 245, 0.3);
}}
"""
```

### 性能优化
- 使用 `setAttribute(Qt.WidgetAttribute.WA_StyledBackground)`
- 避免频繁的 `setStyleSheet` 调用
- 使用 `QGraphicsDropShadowEffect` 谨慎 (影响性能)
- 日志使用虚拟滚动或分页加载
